# SKALA Portal Hub 고도화 가이드 및 개발 지침 (.prompt.md)

이 문서는 **SKALA Portal Hub** 프로젝트의 프론트엔드 고도화 및 기술 평가(정성적/정량적 평가) 극대화를 위해 AI 어시스턴트(Copilot)가 반드시 준수해야 하는 시스템 프롬프트 및 개발 가이드라인입니다. 코드를 생성하거나 리팩토링할 때 이 규칙을 최우선으로 적용하세요.

---

## 🏛️ 1. 프로젝트 정체성 & 평가 목표

- **프로젝트 명칭**: SKALA Portal Hub (포털 마스터 허브 대시보드)
- **대상 사용자**: 면접관, 멘토, 기술 평가자
- **개발자 정체성**: **지능형 데이터 엔지니어** (AI, 데이터 파이프라인, 자동화, 웰니스 라이프를 결합한 디지털 명함)
- **평가 기준 최적화**:
  - **정량적 평가**: 성능(Lighthouse 최적화, Reflow/Repaint 최소화), 웹 접근성(WAI-ARIA 규격 준수), 반응형 레이아웃의 무결성(모바일/데스크톱 픽셀 완벽 대응).
  - **정성적 평가**: 세련되고 현대적인 인터페이스(Mesh Gradient, Notion Grid, Filter Neon), 외부 라이브러리 없이 구현된 **고수준 Vanilla JS 아키텍처**, 비동기 데이터 처리, 로컬 데이터 영속성 파이프라인.

---

## 🚫 2. 핵심 개발 제약 조건 (Core Constraints)

1. **외부 라이브러리 절대 사용 금지 (No Third-Party Libraries)**:
   - React, Vue, Svelte, Angular 사용 금지.
   - jQuery, Tailwind CSS, Bootstrap, FontAwesome 등 외부 CSS/JS 라이브러리 및 CDN 연동 금지.
   - 오직 **순수 HTML5, Modern CSS3, Vanilla ES6+ JavaScript**로만 모든 컴포넌트와 인터랙티브 기능을 구현합니다.
2. **코드 가독성 및 무결성 (Clean Code)**:
   - 개발 중 임시로 삽입된 불필요한 인라인 주석, 번호 매기기(`[cite: 1]`) 등 가독성을 저해하는 텍스트는 전면 제거합니다.
   - 변수, 함수명은 직관적이고 서술적으로 네이밍합니다. (예: `IntegratedPortalController`, `startHighPerformanceClock`).
3. **Vanilla JS OOP & 클래스 기반 구조**:
   - 상태 관리와 인터랙션은 전역 변수 대신 **클래스 기반으로 캡슐화**하여 네임스페이스 오염을 방지하고 코드의 유지보수성을 극대화합니다.
4. **레이아웃 붕괴 방지 가드레일**:
   - 모든 요소에 `* { box-sizing: border-box; }`를 기본 장착합니다.
   - 반응형 레이아웃 오버플로우 방지를 위해 `overflow-x: hidden`을 전역 컨테이너에 적용하고, 그리드/플렉스 레이아웃의 유연성을 강화합니다.

## 🎨 3. UI/UX 및 디자인 시스템 규격

- **배경 렌더링**: `mesh-gradient-bg`와 `notion-grid-mask`를 조합하여 깊이감 있는 하이테크 감성 연출.
- **테마 엔진 (Multi-Theme)**:
  - `Light`, `Dark`, `Latte` (Oatmeal), `Forest` 4가지 테마를 완벽히 지원합니다.
  - CSS Custom Properties (CSS 변수)를 이용해 테마 변동 시 일관성 있게 색상이 교체되도록 설계합니다.
  - **`filter-neon` 가드레일**: 테마 변경 시에도 하이라이트 발광 효과가 지나치거나 깨지지 않도록 가독성 최적화 값을 유지합니다.
- **공용 헤더 (GNB) 통일**:
  - 모든 서브 페이지(`index.html`, `myProfile.html`, `myClass.html`, `holiday.html`)의 상단에 `🏠 SKALA.Hub` 로고와 '멀티 테마 스위치'를 일관되게 바인딩하여 무결성 있는 네비게이션을 구현합니다.

## 🛠️ 4. 페이지별 고도화 상세 가이드라인

### ① 메인 대시보드 (`index.html`)
- **레이아웃**: 네이버 포털 스타일의 레이아웃(우측 프로필 고정, 좌측 서비스 캐스트 및 알고리즘 실습 샌드박스).
- **실시간 데이터 시각화**: n8n 라이브 스트림 터미널(가상 파이프라인 모니터링 로그)의 동적 렌더링 및 24시간 고성능 시계.
- **알고리즘 샌드박스**: 순수 JS 연산(UpDown 게임, 성적 산출, 가방 채우기 알고리즘 등)을 시각적으로 매끄럽게 처리하는 UX 고도화.
- **[기능 고도화] 조회수 카운터 엔진**:
  - `localStorage`를 활용하여 세션/일자별 중복 방문을 필터링하고 페이지 조회수를 안전하게 증가시키는 자체 카운터 모듈 구축.

### ② 프로필 명세 (`myProfile.html`)
- **컨텐츠 고도화**: 데이터 엔지니어로서의 학력, 수상 실적(Smishing Forecast 등), 기술 스택 명세서.
- **포트폴리오 카드 인터랙션**:
  - 카드를 클릭하면 상세 보기 모달(Modal)이 화면 중앙에 부드럽게 오버레이(Scale & Fade-in)되도록 마이크로 인터랙션 구현.
  - 프로필 사진/이미지는 라이트박스(Lightbox) 모듈을 자체 설계하여 원본 스케일로 왜곡 없이 감상할 수 있게 지원.

### ③ 학사 일정 포털 (`myClass.html`)
- **캘린더 엔진**:
  - 주간(Week), 월간(Month), 일간(Day) 3가지 뷰를 동적으로 스위칭할 수 있는 그리드 기반 캘린더 아키텍처.
- **슬라이드 인포 패널**:
  - 특정 시간표나 과목(n8n, SQL, Git 등)을 클릭하면 우측 또는 좌측에서 부드럽게 나타나는(Slide-in) 상세 정보 패널을 설계하여 '왜 이 기술을 배웠는지(Why)'에 대한 데이터 엔지니어링 철학 표출.

### ④ 일상 로그 & 휴일 루틴 (`holiday.html`)
- **레이아웃**: 네이버 파워블로거 감성을 살린 '주간일기' 스타일 카드 본문.
- **5대 라이프스타일 핫라인**:
  - 러닝, 베이커리, 맛집, 영화, 인형뽑기 등의 단축 칩을 클릭 시, 본문의 해당 섹션으로 `scrollIntoView({ behavior: 'smooth' })` 처리.
  - 스크롤 이동이 끝난 후 해당 카드에 일시적으로 네온 발광 테두리(`box-shadow`) 효과를 주었다가 서서히 사라지게(Fade-out) 만드는 자체 하이라이트 가이드 구현.
- **동적 댓글 CRUD 엔진**:
  - `localStorage`를 활용해 영속성 데이터 파이프라인을 구축.
  - 비동기식(자체 DOM 조작) 댓글 생성, 조회, 수정, 삭제 기능 구현.
  - 수정 및 삭제 시 비밀번호 검증 단계를 추가하여 기능의 완결성 확보.
- **[기능 고도화] 로컬 데이터 Export/Import**:
  - 작성된 방명록/댓글 데이터를 JSON 파일로 다운로드(`Export`)하거나, 저장된 JSON 파일을 불러와 복원(`Import`)할 수 있는 오프라인 데이터 관리 도구 설계.

## 📈 5. 기술적 평가 대비 최적화 규칙 (정량/정성)

### 🚀 정량적 성능 향상 (Lighthouse & Rendering)
- DOM 요소를 빈번히 삭제하고 새로 생성하기보다는 `DocumentFragment`를 사용하거나, `textContent`를 적극 활용하여 브라우저의 Repaint/Reflow 연산 비용을 최소화합니다.
- CSS 애니메이션은 CPU 대신 GPU 가속을 활용하는 `transform`, `opacity` 속성만을 위주로 제어합니다.
- `debounce` 및 `throttle` 헬퍼 함수를 자체 구현하여 스크롤이나 리사이즈 이벤트 시 발생하는 불필요한 JS 연산을 차단합니다.

### ♿ 웹 접근성 (Accessibility)
- 모든 이미지 요소에 구체적인 `alt` 속성을 명시합니다.
- 마우스 없이 키보드 탭 키로도 모든 네비게이션과 모달, 버튼을 제어할 수 있도록 `tabindex`와 `keydown` 이벤트를 바인딩합니다.
- 모달이 활성화되었을 때 초점(Focus)을 모달 내부로 가두는 포커스 트랩(Focus Trap) 메커니즘을 순수 JS로 구현합니다.

### 📱 완전 무결한 반응형 대응
- 모바일(최소 320px)부터 데스크톱(최대 1920px)까지 미디어 쿼리를 설계합니다.
- 모바일 환경에서는 사이드바가 상단/하단의 가로형 스크롤 슬라이더(칩 레일)로 자동 스위칭되어 뷰포트 공간을 효율적으로 활용할 수 있게 합니다.
